594. Longest Harmonious Subsequence

May 20, 2017

Question

We define a harmonious array is an array where the difference between its maximum value and its minimum value is exactly 1.

Now, given an integer array, you need to find the length of its longest harmonious subsequence among all its possible subsequences.

Example 1:

Input: [1,3,2,2,5,2,3,7]
Output: 5
Explanation: The longest harmonious subsequence is [3,2,2,2,3].

Note: The length of the input array will not exceed 20,000.

Solution

Approach #1 Brute Force [Time Limit Exceeded]

In the brute force solution, we consider every possible subsequence that can be formed using the elements of the given array. For every subsequence, we find the maximum and minimum values in the subsequence. If the difference between the maximum and the minimum values obtained is 1, it means the current subsequence forms a harmonious subsequence. Thus, we can consider the number of elements in this subsequence to be compared with the length of the last longest harmonious subsequence.

In order to obtain all the subseqeuences possible, we make use of binary number representation of decimal numbers. For a binary number of size nnn, a total of 2n2^n2n different binary numbers can be generated. We generate all these binary numbers from 000 to 2n2^n2n. For every binary number generated, we consider the subsequence to be comprised of only those elements of numsnumsnums which have a 1 at the corresponding position in the current binary number. The following figure shows an example of the way the elements of numsnumsnums are considered in the current subsequence.

Java

public class Solution {
    public int findLHS(int[] nums) {
        int res = 0;
        for (int i = 0; i < (1 << nums.length); i++) {
            int count = 0, min = Integer.MAX_VALUE, max = Integer.MIN_VALUE;
            for (int j = 0; j < nums.length; j++) {
                if ((i & (1 << j)) != 0) {
                    min = Math.min(min, nums[j]);
                    max = Math.max(max, nums[j]);
                    count++;
                }
            }
            if (max - min == 1)
                res = Math.max(res, count);
        }
        return res;
    }
}

Complexity Analysis

  • Time complexity : O(2n)O(2^n)O(2n). Number of subsequences generated will be 2n2^n2n.

  • Space complexity : O(1)O(1)O(1). Constant space required.

Approach #2 Better Brute Force [Time Limit Exceeded]

Algorithm

In the last approach, we created every possible subsequence, and for every such subsequence, we found out if it satisfies the harmonicity condition. Instead of doing this, we can do as follows. We can consider every element of the given numsnumsnums array one by one. For nums[i]nums[i]nums[i] chosen to be the current element, we determine the countcountcount of all the elements in the numsnumsnums array, which satisfy the harmonicity condition with nums[i]nums[i]nums[i], i.e. the countcountcount of all such nums[j]nums[j]nums[j] satisfying nums[i]==nums[j]nums[i] == nums[j]nums[i]==nums[j] or nums[i]==nums[j]+1nums[i] == nums[j] + 1nums[i]==nums[j]+1. When we reach the end of the array for nums[i]nums[i]nums[i] being the current element, we compare this countcountcount obtained with the result obtained from the previous traversals and update the result appropriately. When all the elements of the array have been chosen as the element to be chosen as the base for harmonicity check, we get the required length of the longest harmonic subsequence.

public class Solution {
    public int findLHS(int[] nums) {
        int res = 0;
        for (int i = 0; i < nums.length; i++) {
            int count = 0;
            boolean flag = false;
            for (int j = 0; j < nums.length; j++) {
                if (nums[j] == nums[i])

                else if (nums[j] + 1 == nums[i]) {

Approach #3 Using Sorting [Accepted]

Algorithm

Since we are concerned only with the count of elements which are at a difference of 1, we can use sorting to our advantage. If we sort the given numsnumsnums array, the related elements will get arranged close to each other. Thus, we can traverse over the sorted array, and find the count of similar elements and elements one larger than the current ones, which occur consecutively(all the similar elements will be lying consecutively now). Initially, this value is stored in prev_countprev\_countprev_count variable. Then, if we encounter an element which is just 1 larger than the last elements, we count the occurences of such elements as well. This value is stored in countcountcount variable.

Thus, now for the harmonic subsequence comprised of only these two elements is a subsequence of length count+prev_countcount + prev\_countcount+prev_count. This result is stored in resresres for each subsequence found. When we move forward to considering the next set of similar consecutive elements, we need to update the prev_countprev\_countprev_count with the countcountcount's value, since now countcountcount will act as the count of the elements 1 lesser than the next elements encountered. The value of resresres is always updated to be the larger of previous resresres and the current count+prev_countcount + prev\_countcount+prev_count value.

When we are done traversing over the whole array, the value of resresres gives us the required result.

Java

public class Solution {
    public int findLHS(int[] nums) {
        Arrays.sort(nums);
        int prev_count = 1, res = 0;
        for (int i = 0; i < nums.length; i++) {
            int count = 1;
            if (i > 0 && nums[i] - nums[i - 1] == 1) {
                while (i < nums.length - 1 && nums[i] == nums[i + 1]) {
                    count++;
                    i++;
                }
                res = Math.max(res, count + prev_count);
                prev_count = count;
            } else {
                while (i < nums.length - 1 && nums[i] == nums[i + 1]) {
                    count++;
                    i++;
                }
                prev_count = count;
            }
        }
        return res;
    }
}

Complexity Analysis

  • Time complexity : O(nlogn)O(nlogn)O(nlogn). Sorting takes O(nlogn)O(nlogn)O(nlogn) time.

  • Space complexity : O(logn)O(logn)O(logn). lognlognlogn space is required by sorting in average case.

Approach #4 Using HashMap[Accepted]:

Algorithm

In this approach, we make use of a hashmap mapmapmap which stores the number of times an element occurs in the array along with the element's value in the form (num:countnum)(num: count_num)(num:countnum), where numnumnum refers to an element in the array and countnumcount_numcountnum refers to the number of times this numnumnum occurs in the numsnumsnums array. We traverse over the numsnumsnums array and fill this mapmapmap once.

After this, we traverse over the keys of the mapmapmap created. For every key of the mapmapmap considered, say keykeykey, we find out if the map contains the key+1key + 1key+1. Such an element is found, since only such elements can be counted for the harmonic subsequence if keykeykey is considered as one of the element of the harmonic subsequence. We need not care about key1key - 1key1, because if keykeykey is present in the harmonic subsequence, at one time either key+1key + 1key+1 or key1key - 1key1 only could be included in the harmonic subsequence. The case of key1key - 1key1 being in the harmonic subsequence will automatically be considered, when key1key - 1key1 is encountered as the current key.

Now, whenver we find that key+1key + 1key+1 exists in the keys of mapmapmap, we determine the count of the current harmonic subsequence as countkey+countkey+1count_{key} + count_{key+1}countkey+countkey+1, where counticount_icounti refers to the value corresponding to the key iii in mapmapmap, which reprents the number of times iii occurs in the array numsnumsnums.

Java

public class Solution {
    public int findLHS(int[] nums) {
        HashMap < Integer, Integer > map = new HashMap < > ();
        int res = 0;
        for (int num: nums) {
            map.put(num, map.getOrDefault(num, 0) + 1);
        }
        for (int key: map.keySet()) {
            if (map.containsKey(key + 1))
                res = Math.max(res, map.get(key) + map.get(key + 1));
        }
        return res;
    }
}

Complexity Analysis

  • Time complexity : O(n)O(n)O(n). One loop is required to fill mapmapmap and one for traversing the mapmapmap.

  • Space complexity : O(n)O(n)O(n). In worst case map size grows upto size nnn.

Approach #5 In single loop [Accepted]:

Algorithm

Instead of filling the mapmapmap first and then traversing over the mapmapmap to determine the lengths of the harmonic subsequences encountered, we can traverse over the numsnumsnums array, and while doing the traversals, we can determine the lengths of the harmonic subsequences possible till the current index of the numsnumsnums array.

The method of finding the length of harmonic subsequence remains the same as the last approach. But, this time, we need to consider the existence of both key+1key + 1key+1 and key1key - 1key1 exclusively and determine the counts corresponding to both the cases. This is needed now because it could be possible that keykeykey has already been added to the mapmapmap and later on key1key - 1key1 is encountered. In this case, if we consider the presence of key+1key + 1key+1 only, we'll go in the wrong direction.

Thus, we consider the countcountcounts corresponding to both the cases separately for every keykeykey and determine the maximum out of them. Thus, now the same task can be done only in a single traveral of the numsnumsnums array.

Java

public class Solution {
    public int findLHS(int[] nums) {
        HashMap < Integer, Integer > map = new HashMap < > ();
        int res = 0;
        for (int num: nums) {
            map.put(num, map.getOrDefault(num, 0) + 1);
            if (map.containsKey(num + 1))
                res = Math.max(res, map.get(num) + map.get(num + 1));
            if (map.containsKey(num - 1))
                res = Math.max(res, map.get(num) + map.get(num - 1));
        }
        return res;
    }
}

Complexity Analysis

  • Time complexity : O(n)O(n)O(n). Only one loop is there.

  • Space complexity : O(n)O(n)O(n). mapmapmap size grows upto size nnn.
